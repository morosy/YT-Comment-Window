# Skill: YouTube DOM Strategy

## 前提
YouTube は DOM 構造が変化しやすく、さらに SPA 遷移と遅延描画がある。
そのため、静的な1回探索だけで完結させない。

## 方針
1. 主要コンテナを探索する
2. コメント領域を探索する
3. 右カラム近辺の配置先を探索する
4. 見つからなければ一定時間リトライする
5. それでも見つからなければ安全に中断する

## 実装ポイント
- セレクタは1か所にまとめる
- `MutationObserver` で変化監視する
- 二重挿入を防ぐため、拡張機能ルート要素に一意 ID を付ける
- 元要素を戻せるよう、元の親要素と兄弟位置を保持する

## 初版で優先する安全策
- コメント本体を丸ごと複製しない
- ページ全体の CSS を広範囲に上書きしない
- 右カラムを完全に消さない

## 失敗時
- `console.warn` で理由を出す
- ユーザー向けには popup 側で簡潔に失敗を伝えられる余地を残す
