# PLANS.md

## 1. 目的
YouTube の動画視聴中に、ページ全体を縦スクロールしなくてもコメントを同時閲覧できる Chrome 拡張機能を開発する。

現状の YouTube では、動画とコメント欄が縦方向に離れており、同一画面で快適に見にくい。
本拡張機能では、コメント欄を右側の簡易ウィンドウとして表示し、ページ本体は動画中心のまま維持する。

## 2. 初版スコープ
### 必須
- YouTube の通常動画ページで動作する。
- 拡張機能アイコンをクリックして popup を開ける。
- popup の `コメント欄をウィンドウ化` ボタン押下で処理開始できる。
- コメント欄を右側へ独立スクロール可能な UI として表示できる。
- 閉じる、もしくは元に戻すことができる。

### 初版では見送るもの
- ドラッグ移動
- 自由リサイズ
- テーマ切替
- 設定画面
- ショートカットキー
- 複数レイアウトモード
- コメントの抽出・検索・フィルタ
- YouTube Shorts / Live / 埋め込み再生への最適化

## 3. 想定利用フロー
1. ユーザーが YouTube の動画ページを開く。
2. 拡張機能アイコンをクリックする。
3. popup の `コメント欄をウィンドウ化` を押す。
4. 動画右側にコメント用ウィンドウが表示される。
5. ユーザーは動画を見ながら、コメントウィンドウのみをスクロールして閲覧する。
6. 必要なら閉じる／元に戻す。

## 4. 機能要件
### 4.1 popup
- 最低限の説明を表示する。
- 実行ボタンを1つ持つ。
- YouTube watch ページ以外では、無効状態または案内文を表示してもよい。

### 4.2 コメントウィンドウ化
- 右側カラム近辺に表示する。
- ウィンドウ内部だけがスクロールする。
- ヘッダー部にタイトルと閉じるボタンを置く。
- 可能であればコメント領域本体を再配置し、難しければ安全なラッパー化を行う。

### 4.3 復元
- ウィンドウ化前の状態へ戻せること。
- 二重起動して DOM が破損しないこと。

### 4.4 YouTube SPA対応
- YouTube は SPA 遷移を行うため、動画切替時に壊れにくいこと。
- 画面遷移後、必要に応じて UI 状態を破棄／再生成できること。

## 5. 非機能要件
- 外部サーバー不要。
- 個人情報の外部送信なし。
- 低依存・低複雑性。
- YouTube 側仕様変更時に追従しやすい分割構成。

## 6. 技術設計方針
### 6.1 拡張機能構成
- `manifest.json`: MV3 構成。MV3 は現行の Chrome Extensions 基盤。citeturn239732search1turn239732search8
- `popup/`: ユーザー起点 UI
- `src/background/service-worker.js`: popup からの命令受け、タブ操作、必要に応じたメッセージ中継
- `src/content/`: YouTube ページ DOM 操作用

### 6.2 権限方針
初版候補:
- `activeTab`
- `scripting`
- `storage`（必要になれば）
- `host_permissions`: `https://www.youtube.com/*`

`chrome.scripting` の利用には `scripting` 権限と対象ページ権限が必要。citeturn239732search0

### 6.3 DOM戦略
優先順位は以下。
1. **既存コメント要素を安全に再配置する**
2. それが困難なら、元コメントをラップしつつ見た目上右カラムへ表示する
3. 最後の手段として、複製ビューを検討する

初版では 1 または 2 を目指す。

### 6.4 状態管理
初版の状態は少ないため、まずは軽量な in-memory 管理とする。
想定 state:
- isWindowMode
- originalParent
- originalNextSibling
- overlayRoot
- currentVideoId

## 8. 受け入れ基準
- unpacked load でエラーなく読み込める。
- watch ページ以外で暴走しない。
- watch ページで popup ボタンから機能起動できる。
- コメント欄を独立スクロールできる。
- 動画プレイヤーが上部で視認できる。
- 閉じる操作で元に戻せる。
- 同一ページで複数回操作しても破綻しない。
- 動画を切り替えても重大エラーになりにくい。

## 9. 手動テスト観点
- 通常動画ページで動くか
- コメント数が多い動画でスクロールできるか
- コメント読み込み前に起動しても壊れないか
- プレイリスト付き動画でレイアウト崩れがないか
- 横幅が狭いブラウザで極端に壊れないか
- 閉じる→再起動で戻せるか
- 別動画へ遷移後に再実行できるか

## 10. 既定UI仕様（暫定）
### popup
- タイトル: `YT-Comment-Window`
- 説明: `YouTube のコメント欄を右側のウィンドウに表示します。`
- 主ボタン: `コメント欄をウィンドウ化`

### コメントウィンドウ
- ヘッダー左: `コメント`
- ヘッダー右: `閉じる`
- 本文: YouTubeコメント領域

## 11. 未確定事項
以下は発注者確認待ちだが、初版では暫定値で実装してよい。
1. **ウィンドウ幅**
   - 暫定: 360px 前後
2. **最小ブラウザ幅時の挙動**
   - 暫定: 横幅不足時はウィンドウ幅を縮小し、極端な場合は起動しない
3. **元のコメント欄の扱い**
   - 暫定: 元位置から見えなくする / 再配置する
4. **プレイリストとの共存**
   - 暫定: 初版では完全最適化までは求めない
5. **起動状態の保持**
   - 暫定: ページ遷移やリロード後は解除状態に戻す
6. **右カラムのどの位置に差し込むか**
   - 暫定: 関連動画エリアより優先して上部表示

## 12. 発注者への確認質問案
以下は次回確認候補。
- コメントウィンドウは固定幅で良いか。
   →  スクリーンサイズやブラウザ幅に応じて柔軟に変化させるべき．
      YouTubeの右カラムの幅に合わせるのが自然か．
      マウスのドラッグでも調整できるようにする．
- プレイリスト表示時は、コメントを優先するか、共存を重視するか。
   →  拡張機能起動時はコメントウィンドウを優先して上部に表示する。
      本体のウィンドウをクリックしたとき，本体ウィンドウが上に来るようにする．
      逆にコメントウィンドウをクリックしたときはコメントウィンドウが上に来るようにする．
      コメントウィンドウを見失ったときの為，popupに「ウィンドウを前面に出す」ボタンを置く．
- 右カラムに「重ねる」のか、「置き換える」のか、どちらを好むか。
   →  YouTube本体のウィンドウに重ねるように新しいウィンドウを表示される．
      右カラムのスペースを占有するのではなく，動画プレイヤーの上に重ねる形で表示する．
      これにより，動画とコメントの両方が同時に見やすくなる．
- 動画切替後も自動で再適用したいか。
   →  動画切替後も自動で再適用する．
      ただし，ユーザーが明示的にウィンドウ化を解除した場合（ウィンドウを閉じる）は，次の動画でも自動でウィンドウ化しないようにする．
- 将来的にドラッグ移動やサイズ変更を入れたいか。
   →  ドラッグ移動やサイズ変更を実装する．
      ただし，ドラッグ移動後の位置やサイズはページ遷移やリロード後も保持しない．

## 13. Codexへの実装姿勢
- 初版は **「YouTubeの右側に独立スクロールコメント領域を作る」** ことだけに集中する。
- 難所に遭遇したら、無理に高度化せず、より壊れにくい代替案を選ぶ。
- YouTube DOM 依存部分はコメントで理由を残す。
